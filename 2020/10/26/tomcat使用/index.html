Tomcat使用

Tomcat🦁 服务器是一个免费的开放源代码的Web 应用服务器，属于轻量级应用服务器，在中小型系统和并发访问用户不是很多的场合下被普遍使用，是开发和调试JSP 程序的首选。

安装

进入Apache官网下载Tomcat解压即可，解压后安装目录的文件结构如下：

启动✅

双击bin目录下的start.bat文件，可以启动Tomcat。

打开浏览器输入：http://localhost:8080，即可看到Tomcat正常启动后的管理控制台页面。

关闭❌

双击bin目录下的shutdown.bat文件，可以关闭Tomcat。如果直接关闭运行窗口，会出现Tomcat不能及时释放资源的问题。

或者ctrl + c

部署项目📂

方式一

直接将项目放到webapps目录下

例如在webapps目录新建一个hello文件夹，文件夹内有一个hello.html文件，则启动服务器后，通过http://localhost:8080/hello/hello.html即可访问

简化部署：将项目打包成一个war包，再将war包放到webapps目录下，war包会自动解压缩成目录

方式二

配置conf/server.xml文件

在<host>标签体中设置<Context docBase="D:\hello" path="/he" />

  • docBase：项目存放路径
  • path：虚拟目录

方式三⭐（热部署）

在conf\Catalina\localhost创建任意名称的xml文件。

在文件中编写<Context docBase="D:\hello" />，虚拟目录：xml文件的名称

Java动态项目目录结构

  • 项目根目录
    • WEB-INF目录
      • web.xml：web项目核心配置文件
      • classes目录：放置字节码文件
      • lib目录：防止依赖的jar包

IDEA部署

选择run菜单下的Edit Configurations

选择Defaults下的Tomcat Server->Local

点击Configure，选择Tomcat的目录，完成配置

创建Project/Module：

  • 选择Java Enterprise ，选择version，选择Server为Tomcat
  • 勾选Web Application
  • 编写项目，启动即可

热部署：

  • 选择run菜单下的Edit Configurations
  • 选择Tomcat Server ，在Server修改：
    • On ‘Update’ action：Update resources
    • On frame deactivation Update resources
  • 修改配置后，不需重启服务器即可完成部署
