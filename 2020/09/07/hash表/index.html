hash表

哈希表

根据设定的哈希函数 H(key) 和处理冲突的方法将一组关键字映像到一个有限的地址集上，并以关键字在地址集中的“像”作为记录在表中的存储位置，这种表成为哈希表。

例如，取首字母序号作为hash函数

【key】 BEIJING TIANJIN HEBEI SHANGHAI
【f(key)】 02 20 08 19

可见：

  • hash函数是一个映像，只要使得任何 key 由函数所得的 f(key)落在表长允许的范围内

  • 冲突现在：不同的关键字得到同一个函数值

  • ⚡冲突只能尽可能少，而不能完全避免⚡

    集合包括：

    • 关键字集合：包括所有可能的关键字
    • 地址集合：哈希表的地址值

hash函数构造

均匀hash函数：关键字集合中的任一关键字，经hash函数映像到地址集合中的任意地址概率相等。

构造方法：

  • 直接定址法：取关键字或某个线性函数值为哈希地址。

    H (key) = key 或 H (key) = a * key +b【自身函数】

    💎地址集合与关键字集合大小相同，通常不同关键字不会发生冲突，但很少使用。

    例如：1-100岁的人口数统计表，年龄作k，h(key)=key

    例如：解放后每年的人口调查表，年份作key，h(key)=key+(-1948)

  • 数字分析法：假设关键字以r为基，且hash表中可能出现的关键字事先知道，则可取关键字的若干位组成hash地址

  • 平方取中法：取关键字平方后的中间几位为哈希地址。

    💎较为常用，取的位数由表长决定。

  • 折叠法：将关键字分割为位数相同的几部分（最后一部分位数可不同），然后取这几部分的叠加和作为哈希地址。

    • 移位叠加：将分割后的每一部分的最低位对齐，然后相加
    • 间界叠加：从一端向另一端沿分割界来回折叠，然后对齐相加

    例如，图书编号0-442-20586-4

    移位叠加：5864+4220+04=10088 H(key)=0088

    间界叠加：5864+0224+04=6092 H(key)=6092

  • 除留余数法：取关键字背某个不大于哈希表表长m的数p除后所得余数为哈希地址

    H(key) = key MOD p , p≤m

    💎最简单，最常用的方法，一般情况下，可以选p为质数或不包含小于20的质因数的合数

  • 随机数法：选择一个随机函数，取关键字的随机函数值为它的哈希地址
    H(key) = random(key)

    💎当关键字长度不等时采用此法构造hash函数

处理冲突的方法

开放定址法

Hi = ( H(key) + di) MOD m i=1,2,…,k (k≤m-1)

其中，H(key)为hash函数，m为哈希表表长，di为增量序列，一般有下列三种取法：

  1. 线性探测再散列：di = 1，2，3，…，m-1
  2. 二次探测再散列：di = 1^2，-1^2，2^2，-2^2，…，±k^2 （k≤m/2）
  3. 伪随机探测再散列：di = 伪随机数序列

再哈希法

Hi = RHi(key) i = 1，2，…，k

RHi均是不同的hash函数，及同义词产生地址冲突时计算另一个哈希函数地址，直到冲突不再发生。

💎该方法不易产生“聚集”，但增加了计算时间
