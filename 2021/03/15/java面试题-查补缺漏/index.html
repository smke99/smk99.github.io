关键字

final关键字

  • final修饰变量，如果是基本数据类型的变量，则其数值一旦在初始化之后便不能更改，如果是引用类型，则在对其初始化之后便不能再让其指向另一个对象。
  • final修饰方法，表示方法不可重写。
  • final修饰类，表示该类不可继承。

finally

  • 异常处理，try/catch语句中，在try语句得到执行后，finally语句才能执行。

finalize

  • 方法名，finalize（）方法在垃圾收集器将对象从内存中清除出去之前做必要的清理工作。

Java8新特性

  • Lambda 表达式 − Lambda允许把函数作为一个方法的参数（函数作为参数传递进方法中。
  • 方法引用− 方法引用提供了非常有用的语法，可以直接引用已有Java类或对象（实例）的方法或构造器。与lambda联合使用，方法引用可以使语言的构造更紧凑简洁，减少冗余代码。
  • 默认方法− 默认方法就是一个在接口里面有了一个实现的方法。
  • 新工具− 新的编译工具，如：Nashorn引擎 jjs、 类依赖分析器jdeps。
  • Stream API −新添加的Stream API（java.util.stream） 把真正的函数式编程风格引入到Java中。
  • Date Time API − 加强对日期与时间的处理。
  • Optional 类 − Optional 类已经成为 Java 8 类库的一部分，用来解决空指针异常。
  • Nashorn, JavaScript 引擎 − Java 8提供了一个新的Nashorn javascript引擎，它允许我们在JVM上运行特定的javascript应用。

面向对象

请解释hashCode()和equals()方法有什么联系

  • Java对象的eqauls方法和hashCode方法是这样规定的：

    ➀相等（相同）的对象必须具有相等的哈希码（或者散列码）。

    ➁如果两个对象的hashCode相同，它们并不一定相同。

Java中的方法覆盖(Overriding)和方法重载(Overloading)是什么意思

  • 重载：指同一个类 多个方法 方法名相同 参数不同的情况【返回值一样，如果不一样不算重载】
  • 覆盖：子类重新定义了父类方法，覆盖必须方法名相同 ，参数相同，返回值相同

接口和抽象类

  • 抽象类：不能实例化，如果一个类有一个抽象方法，那么这个类一定是抽象类；反之，抽象类中可以没有抽象方法。一个类继承抽象类，子类必须实现父类的抽象方法，且只能extends继承一个抽象类。（如果子类没有实现，那么子类也必须定义为抽象类）
  • 接口：不能实例化，接口中的方法必须都是抽象方法。一个类可以implements实现多个接口，实现类必须实现所有的抽象方法。
  • 区别：
    • 抽象类可以提供成员方法的实现细节，而接口中只能存在public abstract 方法；
    • 抽象类中的成员变量可以是各种类型的，而接口中的成员变量只能是public static final类型的；
    • 接口中不能含有静态代码块以及静态方法，而抽象类可以有静态代码块和静态方法；
    • 一个类只能继承一个抽象类，而一个类却可以实现多个接口。
    • 抽象类是对一种事物的抽象，即对类抽象，而接口是对行为的抽象。

static

  • 一个成员变量或成员方法，在没有所属的类的实例时，可以被访问

集合

数组(Array)和列表(ArrayList)的区别？

  • Array可以包含基本类型，对象类型，大小固定
  • ArrayList只能包含对象类型，大小动态变化，且添加了更多的方法和特性

介绍集合：

两大接口:Collection,Map接口，Queue

  • 首先，Collection接口定义单列集合，又分为List接口和Set接口。
    • List集合是线性存储的有序集合，元素可以重复，可以通过索引访问
      • ArrayList：数组存储，查找速度快
      • Vector：数组存储，线程安全，但是性能比较差，不经常使用
      • LinkedList：双向链表存储，增删速度快，查找慢，
    • Set集合是无需集合，元素不可用重复
      • HashSet，用hash值确定存储位置，存取和查找性能高。子类LinkedHashSet，链表和哈希表组合的一个数据存储结构，保证元素有序
      • TreeSet，元素有序
  • Map接口定义的是双列集合，采用键值对映射，map集合中key不能相同，每个key映射一个value
    • HashMap，采用哈希表存储，存取顺序不一致。子类是LinkedHashMap，使用链表+哈希表存储，可以保证元素存储顺序一致，
    • TreeMap，采用红黑树存储，存储顺序一致。

多线程

run()和start()的区别：

  • run()是一个普通方法，执行run后程序依然只有主线程这一个线程，程序的执行路径还是一条
  • start()是启动线程的方法，执行start会启动一个线程，该线程处于就绪状态，然后调用run执行线程体，run方法运行结束，线程终止，cpu运行其他线程

wait()和sleep()和yield()：

  • wait：Object对象的方法，调用会放弃对象锁，进入等待队列。只能在同步方法或块中使用，调用notify()/notifyAll()唤醒指定的线程或者所有线程
  • sleep：线程类的方法，使一个线程处于睡眠，指定时间后恢复，进入阻塞状态。可以在任何地方使用。
  • yield：线程让步，执行后进入就绪状态，让自己或者优先级更高的的线程运行

实现线程：

  • 继承 Thread 类
  • 实现 Runnable 接口
  • 实现 Callable接口

实现同步：

  • synchronized关键字

  • synchronized修饰方法

  • Lock锁

    • public void lock() :加同步锁。
    • public void unlock() :释放同步锁。

SQL

SQLite，Mysql，Oracle数据库的比较

  • sqlite，轻量级小型数据库，功能较少，但是更快，适用于100w一下的数据量，不支持分布式。一般用于单机数据存储，方便小型程序使用。
  • mysql，开源免费数据库，配置简单，一般用于中小型网站，但是安全性较差，没有恢复数据的服务。在innodb存储的行级锁的情况下才可支持事务，不支持行级锁和外键。
  • Oracle，收费，使用授权表管理用户权限，一般用于金融/电信等对可靠性高的企业，提供行级锁和外键约束，提供数据库ACID事务的支持。
